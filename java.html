Présentation du projet E5

Description des Fonctionnalités et Caractéristiques de l'Application

Caractéristiques Générales :

Interface en Ligne de Commande : L'application est actuellement accessible uniquement via une interface en ligne de commande.

Architecture 3-Tiers : Conçue selon une architecture 3-tiers, l'application doit conserver cette structure applicative.

Multi-Utilisateurs : L'application doit être adaptée pour supporter plusieurs utilisateurs grâce à l'intégration d'une base de données.

Niveaux d'Habilitation des Utilisateurs :

Employé de Ligue :

  • Consultation : Peut ouvrir l'application et l'utiliser comme un annuaire.
  • Restrictions : Ne dispose d'aucun droit d'écriture.

Administrateur de Ligue :

  • Gestion des Employés : Dispose de droits d'écriture et peut gérer la liste des employés de sa propre ligue.
  • Interface : Utilise une application de bureau pour effectuer ces tâches.

Super-Administrateur :

  • Accès Global : A accès en écriture à tous les employés des ligues.
  • Gestion des Comptes : Peut gérer les comptes des administrateurs des ligues.
  • Interface : Utilise une application accessible en ligne de commande.

Fonctionnalités Rajoutée :

Saisie de date d'inscription d'employés en ligne de commande : Chaque administrateur de ligue doit pouvoir recenser les employés de sa ligue.

Mise en Place des Niveaux d'Habilitation : Il existe trois niveau d'Habilitation, le Super- administrateur qui accès a tous les droit, l'administrateur de ligue qui peut gérer sa ligue et l'employé qui a juste un droit de lecture sur l'application.
